# 02. 式と関数

この章では、Sapphire の中で **値を作る・名前を付ける・関数を
呼ぶ** までの最小限の道具立てを揃える。Ruby を書ける人が読み
始めても、最初の一段は地続きに見える層である。

主に文書 01（コア式言語）、02（字句構文）、05（演算子と数値）
を背景にしている。

## まずは値とリテラル

Sapphire のコア言語が最初から知っている型は次の三つ
（文書 01 §Types）。

```
42          -- Int
"hello"     -- String
True        -- Bool（実体は data ADT、文書 09）
False       -- Bool
```

整数リテラルは `1_000_000` のように `_` で桁区切りできる
（文書 02 §Integer literals）。文字列は `"..."` で囲み、
`\n` `\t` `\\` `\"` `\u{1F600}` といったエスケープが使える。

Ruby との差で先に知っておくとよい点:

- 整数の頭に `-` をつけて `-3` と書ける**ように見える**が、
  これは「単項マイナス」という前置演算子が `negate 3` の糖衣
  になっているだけで、リテラル自体に符号は含まれない（文書 05
  §Unary minus）。
- 文字列の中に式を埋め込む `"#{name}"` のような補間は **まだ
  仕様にない**。`"Hello, " ++ name` のように `++` で連結する。

## 名前をつける: `let`

ローカルに値を束縛するには `let` を使う。

```
let x = 1 in
let y = 2 in
x + y
```

`let 名前 = 式 in 式` という一つの大きな式で、`in` の右側に
書いた式の値が全体の値になる。Ruby のメソッド本体での

```ruby
x = 1
y = 2
x + y
```

に対応するが、Sapphire では「行が並んでいる」のではなく
「式が入れ子になっている」点が違う。

トップレベル（モジュールのファイル直下）では `let ... in` の
代わりに、ただ

```
x : Int
x = 1

y : Int
y = 2
```

のように書く。トップレベルの定義は順序によらず相互に参照できる
ようになっている（文書 03 §Top-level declarations）。

## 条件式: `if ... then ... else`

`if` は文ではなく式である。両方の腕が同じ型を返さなければなら
ない。

```
let abs = \n -> if n < 0 then -n else n in
abs (-5)        -- 5
```

Ruby の三項演算子 `n < 0 ? -n : n` がいちばん近い。Ruby のように
`if` を文として使い、副作用を起こして抜ける、という書き方は
Sapphire には存在しない。

なお `if` は文書 09 で「`case c of { True -> t ; False -> f }` の
糖衣」と再定義されている。`Bool` がただの ADT として定義されて
いるからである（後の章で扱う）。

## 関数を作る: ラムダと関数定義

`\param -> body` がラムダ式である。Ruby の `->(param) { body }`
にほぼ対応する。

```
let inc = \n -> n + 1 in
inc 41          -- 42
```

引数を複数取りたいときはラムダを入れ子にする。

```
let add = \a -> \b -> a + b in
add 3 4         -- 7
```

これは何度も書くと面倒なので、トップレベルや型クラスのメソッド
定義では **clause 形式** が使える（文書 07 §Abstract syntax の
`clause` 形）。

```
add : Int -> Int -> Int
add a b = a + b
```

これは内部的に `add = \a -> \b -> a + b` と同じである。

### 部分適用とカリー化

`add` の型は `Int -> Int -> Int` であり、これは
`Int -> (Int -> Int)` の意味（`->` は右結合、文書 01 §Types）。
すなわち「Int を一つ受け取って、Int を受け取って Int を返す関数
を返す」関数である。

```
let inc = add 1 in
inc 10          -- 11
```

`add 1` だけを評価するとそれ自体が「`Int -> Int` 型の値」で
ある。Ruby の `method(:add).curry[1]` を常に勝手にやってくれる、
くらいに考えてよい。

### 関数の適用は最強

`f x y` は `(f x) y` と読む（左結合）。`f 3 + g 5` は
`(f 3) + (g 5)` と読む（関数適用が `+` よりも強く結合する。
文書 05 §Operator table）。

つまり Ruby の `f(3, 5)` に当たるのは `f 3 5` であり、`,` は
区切り文字としては出てこない。

## 演算子表

文書 05 が定めている主な二項演算子をまとめると:

| 段 | 結合性 | 演算子 | 型（M5 時点） |
|----|--------|--------|----------------|
| 7 | 左 | `*` `/` `%` | `Int -> Int -> Int` |
| 6 | 左 | `+` `-` | `Int -> Int -> Int` |
| 5 | 右 | `++` | `String -> String -> String` |
| 5 | 右 | `::` | リスト cons（後の章） |
| 4 | なし | `==` `/=` `<` `>` `<=` `>=` | 比較（型クラスで一般化、後の章） |
| 3 | 右 | `&&` | `Bool -> Bool -> Bool` |
| 2 | 右 | `\|\|` | `Bool -> Bool -> Bool` |
| 1 | 左 | `>>=` `>>` | モナド系（後の章） |

注意点:

- 段 4 の比較は **非結合**。`a < b < c` は構文エラーになる
  （文書 05 §Operator table）。Ruby のように連鎖比較は許されない。
- ユーザーが自前の優先順位を宣言する仕組み（Haskell の
  `infixl N`）はない（少なくとも今のドラフトでは）。
- `+` `-` は数値専用。文字列連結は `++`。

演算子を **関数として** 渡したいときは括弧で囲む。

```
let plus = (+) in
plus 1 2        -- 3
```

これも文書 05 が許していて、`(+)` は普通の `Int -> Int -> Int`
として通用する。

## 型シグネチャ

トップレベルの定義には型シグネチャを併記できる。

```
double : Int -> Int
double n = n + n
```

文書 08（モジュール）が定めるルールでは、**他のモジュールに
公開する** トップレベル束縛にはシグネチャが **必須** であり、
そのモジュールの中だけで使うものは省略可能（型推論される）。
チュートリアルの段では「公開関数には書く」とだけ覚えておけば
十分である。

シグネチャの中の小文字の名前は **型変数**（多相な型を表す）、
大文字の名前は **型コンストラクタ**（`Int`、`String`、`Maybe`
など）。

```
id : a -> a
id x = x

const : a -> b -> a
const x _ = x
```

`id` は「どんな型 `a` についても、`a` を受け取って `a` を
返す」関数である。`const` の二番目の引数 `_` は「使わない」
ことを表すワイルドカード（文書 02 §Identifiers）。

Ruby に強制しようがない概念として、Sapphire のコンパイラは
「`id 1` のときは `a = Int`、`id "x"` のときは `a = String`」
を **定義の場所で一度に推論して、使い回せる** ようにする
（let-多相、文書 01 §Typing rules）。

## ちょっと複雑な例

ここまでの道具だけで書ける例を一つ。

```
-- 0 から n までの整数の総和
sumTo : Int -> Int
sumTo n =
  let go = \k -> \acc ->
        if k > n then acc else go (k + 1) (acc + k)
  in
  go 0 0
```

`let` で再帰関数 `go` を定義し、それを `0 0` で呼んでいる。
Sapphire の `let` は **暗黙に再帰** であり（文書 03 §Recursive
let）、自分自身を参照できる。

Ruby で書き直すと

```ruby
def sum_to(n)
  go = ->(k, acc) {
    k > n ? acc : go.call(k + 1, acc + k)
  }
  go.call(0, 0)
end
```

で、Ruby ではローカル変数 `go` をラムダ自身が参照できないので
普通は `Method` を使うか、ループで書き直す。Sapphire では再帰が
普通の `let` で素直に書ける。

## レイアウト（軽く）

Sapphire は Haskell や Elm と同じく **インデント感度がある**
（文書 02 §Layout）。ただし本チュートリアルが扱うレベルでは

- `let ... in ...` は同じ行に書ける、
- `case ... of` の腕は同じ列に揃える、

くらいを意識すれば困らない。もし困ったら `{ ... ; ... }` と
波括弧とセミコロンで明示的にブロックを書く逃げ道がある。

```
let { x = 1 ; y = 2 } in x + y
```

## 仕様への気付き

- ここまでの内容は、Ruby しか書いたことのない読者にとっても
  概ね地続きに読めるはず。引っかかりやすいのは「`if` が文で
  はなく式」「整数の `-` は単項演算子で、リテラルの一部では
  ない」「比較は連鎖できない」あたり。
- 文書 01 と 05 のドラフトはこの粒度で読み下せる。`generalize`
  / `instantiate` のような形式記号を出さなくても入門者には
  十分到達可能で、現状の仕様の重さは適切に見える。
