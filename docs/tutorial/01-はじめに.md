# 01. はじめに

このチュートリアルは Sapphire という設計中の関数型言語を、
**手を動かしながら一通り眺める** ためのものである。読者として
想定しているのは、普段 Ruby を書いていて、関数型言語にはわず
かに触れた経験がある人 — つまりこのプロジェクトのオーナー
自身を念頭に書いている。

`docs/spec/` 以下に積まれている 13 本の仕様書は、項書き換え系
の規則や BNF を正面から扱っているため、はじめて読むとどうしても
重く感じる。チュートリアルの目的は、そこに書かれている内容を、
**実際にコードを書くときの順番** に並べ直し、Ruby 利用者の直感に
寄せて説明することである。

## Sapphire とは

ひとことで言えば、

- **Elm のような小ささ・読みやすさ** を出発点にしながら、
- **Haskell 並みの表現力**（型クラスや汎用 `Monad`）を備え、
- **コンパイル先は Ruby** であり、
- 埋め込み Ruby を扱うための専用モナド `Ruby` を言語機能として持つ、

そんな言語として設計が進められている。

`docs/project-status.md` にある通り、現在のフェーズはまだ
**仕様策定段階** であり、コンパイラの実装にも、ホスト言語の
選定にも入っていない。本チュートリアルが扱う Sapphire のコード
は **「仕様書から導かれる、こう書けるはずの形」** であって、
今この場でコンパイルして動かせるものではない。`.sp` ファイルを
書いて `sapphire build` を叩く未来はまだ存在しない。

## 想定読者

- Ruby は普段使っている。`Array`、`Hash`、ブロック、`map` /
  `select` / `inject` などは身近である。
- 静的型はほどほどに知っている（TypeScript や Sorbet 程度）。
- Haskell には昔少しだけ触れたことがあるが、`Monad` や `>>=` の
  気持ちを完全に手の内に入れた感覚は持っていない。

このため、章を進めるごとに **「Ruby で書くとどうなるか」** を
要所で挟みながら、抽象的な概念をなるべく地に足の着いた形で扱う。

## コードサンプルの読み方

各章のコードブロックは、**仕様書のドラフトに基づく説明用の擬似
ソース** であり、実際にコンパイラに通すことはできない。

```
-- これは Sapphire のコメント。
-- 実行できるソースではなく、設計上こう書けるはずという例。
double : Int -> Int
double n = n + n
```

慣例:

- `--` で始まる行は Sapphire のコメント（文書 02 §Whitespace
  and comments）。
- 識別子の頭文字が小文字なら値・関数、大文字なら型・コンストラ
  クタ・モジュール（文書 02 §Identifiers）。
- 型注釈は `名前 : 型` の形で、**定義より先に** あるいは **同じ
  ファイル内のどこかに** 書く（文書 01 §Abstract syntax）。
- 関数定義は `名前 引数1 引数2 = 式` のように複数引数を並べて
  書ける。これは内部的にはラムダの入れ子の糖衣（文書 07 §Abstract
  syntax の `clause` 形）。

Ruby と異なる点で最初に意識するとよいもの:

- **すべては式である。** Ruby も式指向ではあるが、Sapphire は
  さらに徹底していて、`if ... then ... else ...` も `case ... of`
  も値を返す式である（文書 01 §Design notes）。
- **値は基本的にイミュータブル。** 「変数に再代入する」という
  概念は出てこない。`x = ...` は「束縛」であり、上書きではない。
- **副作用は型に現れる。** `puts` を呼ぶような関数は普通の
  `String -> {}` ではなく `String -> Ruby {}` という型を持つ
  （文書 11）。これがチュートリアル後半の鍵になる。

## 仕様書とのつきあい方

各章の本文では、込み入った定義を仕様書から引いてくる代わりに、
たいていの場合は要点の説明と短い例で済ませる。「もう一段詳し
く」という気持ちになったときのために、要所で

> （文書 03 §Typing rules）

のような形で参照を置いておく。仕様書の各文書には章末に
「Open questions」節があり、まだ決まっていない論点がそこに
集まっているので、合わせて眺めると Sapphire が今どこまで
固まっているかが見える。

## このチュートリアルの読み終わり方

06 章まで進むと、

- 純粋な式と関数を書ける、
- `data` で代数的データ型を定義しパターンマッチで分解できる、
- `List` を `map` / `filter` / `foldr` で扱える、
- 自前のモジュールに分割し、`Eq` / `Ord` / `Show` などの型
  クラスを使える、
- `do` 記法で `Maybe` / `Result` / `Ruby` を渡り歩ける、

という地点に立てる。そこから先は実際のプログラム（文書 12
の例題）を読み込むか、仕様書を縦読みして詳細を詰める段階で
ある。

## 次の章では

純粋な部分の最小単位 — リテラル、束縛 (`let`)、条件式
(`if`)、ラムダと関数定義、そして型シグネチャ — を扱う。Ruby と
の見た目の違いと、関数型言語特有の「式が値である」という感覚
にまず慣れてもらうのが狙いである。
