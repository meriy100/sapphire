## 読み方

- 数字は比較の **出発点**。user の価値観（「Ruby 出身者の読みや
  すさを特に重視する」「ADT は死活問題」等）で重みを変えるだけ
  で順位は揺れる。
- 総合 1 位の **Rust** は「コンパイラ実装としての道具立てが最強」
  「target (Ruby) との統合は弱め」のプロファイル。基本点が高く、
  C4 など Ruby 寄り基準に倒しても 1 位を守る（下の感度分析参照）。
- 2 位の **OCaml** は「コンパイラ実装の伝統的な最適解、配布が
  若干弱い」。C9（コミュニティ勢い）で Rust に劣る。
- 3 位の **TypeScript** は C9（コミュニティ勢い）が最上位タイ
  （Rust / Ruby と並ぶ 5）。ただし C3（Ruby 相互運用）で大きく
  失点するため、総合では Rust / OCaml に届かない。
- 4 位の **Haskell** と **Ruby** が同点。意味は全く違う：
  Haskell は C1/C2/C5 で満点、Ruby は C3/C4/C7/C9 で満点。
  user の価値観で大きく動く領域。
- 6 位の **Crystal** は C2 / C9 で失点が重い。Ruby 風の書き味 +
  静的型が欲しい場合の選択肢として残る。

## 重み感度の確認

重みを変えたときの順位変動を検算する。**差分の寄与 = (新重み −
旧重み) × 候補のスコア** で計算する（スコアは §スコア表 を参照）。

### 「Ruby 出身者重視」シナリオ
C4 の重みを ×2 → ×5 に引き上げた場合（差分係数 +3）：

| 候補 | 基本 | C4 スコア | 差分 (+3×score) | 新総合 |
|---|---|---|---|---|
| Rust | 81 | 2 | +6 | **87** |
| Ruby | 69 | 5 | +15 | **84** |
| TypeScript | 71 | 3 | +9 | **80** |
| Crystal | 64 | 5 | +15 | **79** |
| OCaml | 72 | 2 | +6 | **78** |
| Haskell | 69 | 1 | +3 | **72** |

→ **Rust が首位を維持**しつつ、2〜5 位（Ruby / TypeScript /
Crystal / OCaml の順）が僅差で団子になる。「C4 を最重視に倒し
ても Rust が逃げ切る」という結果は、Rust の基本点がもともと高い
ため。

### 「実装品質（C2 + C5）重視」シナリオ
C2 を ×3 → ×5、C5 を ×2 → ×5 に引き上げた場合
（C2 差分係数 +2、C5 差分係数 +3）：

| 候補 | 基本 | C2 / C5 | 差分 (2×C2+3×C5) | 新総合 |
|---|---|---|---|---|
| Rust | 81 | 5 / 5 | +25 | **106** |
| Haskell | 69 | 5 / 5 | +25 | **94** |
| OCaml | 72 | 5 / 4 | +22 | **94** |
| TypeScript | 71 | 4 / 4 | +20 | **91** |
| Ruby | 69 | 3 / 2 | +12 | **81** |
| Crystal | 64 | 2 / 4 | +16 | **80** |

→ **Rust > Haskell = OCaml** の古典的順位。Ruby と Crystal は
実装品質軸に倒すと明確に後退する。
